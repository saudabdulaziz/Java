Lab Assignment Number 3 – cs258


Goals:
1)  Be able to work with individual bits in java.
2)  Serialize objects.
3)  Understand the use of a driver program for ‘white box’ testing.
4)  Develop a program that can create and grade true/false tests.

Synthesis Questions:
1. If x = 37, y=23, z=-110, r=-32 answer the following questions. Assume the variables shown above were declared to be bytes. Show each result in:

·       binary, e.g., 11110000

·       hex, e.g., F0

·       signed decimal, e.g., -16 

You can verify your results by setting the debugger to display hex.  

There will be questions like this on the quiz, so calculate them by hand yourself before verifying them with the debugger (Window | Preferences | Java | Debug | Primitive Display Options | Display Hex values checkbox).


a. System.out.println(x&y);
b. System.out.println(x^y);
c. System.out.println(~x|y);
d. System.out.println(~x>>2);
e. System.out.println((byte)(z<<(~r>>4)));
f. System.out.println
((byte)(x + (x<<1) + (x<<2) + (x<<3)));
g. System.out.println(y>>>3);
h. System.out.println(x&y|z^r);
i. System.out.println(~x + x);
j. System.out.println(x|(3<<4));


2. Assume byte b contains 10110001

a. System.out.println(b) would print what value?

b. Show the mask and operator you would use to set bit 3 in byte b and leave the rest of the bits the same (b ? 0b????????).  

c. Show the mask and operator you would use to clear bit 5 in byte b and leave the rest of the bits the same (b ? 0b????????).  

d. Show the mask and operator you would use to toggle (reverse the value of) bit 2 in byte b and leave the rest of the bits the same (b ? 0b????????).  

 

Requirements:
0) Please put all classes in the default package.


1) The first step is to develop a general-purpose class that will be able to perform operations on strings of bits.
Here is the java doc for Bitmap.java.


Notes:

·       The only instance variable you need is a single long to hold the bits, e.g.,

 long bits;   // this variable must be a long

·       Bits in a BitMap object are numbered from 0 (least significant) to 63 (most significant)

·       Your second and third constructors will be called with, e.g., "ttffft" or the boolean array {true, true, false, false, false, true}

·       Consider making a private method in the BitMap class that returns a long that could be used as a mask by any method performing an operation on a single bit.


While there are many methods specified, the completed class requires only about a page of code.


2)  The next step is to test the BitMap class you created. You can use Driver.java for this purpose.  Don’t modify this program in any way.  It contains a menu that has options to test every BitMap method. Once the testing is complete, BitMap, could be used as a general tool for working with bits and could be used in many programs.


After you have tested your BitMap class, run the set of test cases shown at the bottom of this lab and ensure you get the same answers.


3)  Use notepad or a similar text editor to create a file of true/false questions.  Make sure to have at least 10 questions in your test.


4)  Now write a program (CreateTest.java) that constructs the answer key for a true/false test. This program must:

·       Ask the user for the name of the question file,

·       Read this file, ask the user each question, and record the resulting answers in a bit map object. (Instantiate a BufferedReader and read lines from the text file until you encounter a null line.)

·       Use an ObjectOutputStream to write the BitMap object to a sequential binary file.

·       The name of this file must be the same as the base name of the question file + and have the extension “.key”.


Run this program to create a key file for your test questions.



5)  Finally, create a new application (TakeTest.java). You can copy CreateTest.java to TakeTest.java then modify it appropriately. This program must:

·       ask the user for the name of the question file

·       administer the test to the user and save the user’s answers into a BitMap object

·       read the key file for the question file the user specified (using an ObjectInputStream)

·       compare the answers the user gave to the answers in the key.
display the score earned by the test taker.


In both CreateTest and TakeTest, be sure to handle exceptions appropriately.  Programs should not crash.


6)  Answer the synthesis questions in an rtf or doc file (answers.rtf or answers.doc). Type your name and the lab number on this file and include the questions with the answers.

 
7)  Zip your Eclipse project along with the synthesis answers and submit on moodle.
